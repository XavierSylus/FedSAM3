# Project Rules

- 面向 WPS/Office 使用的 SVG，交付前不能只做浏览器或 XML 合法性校验；必须优先使用 WPS 兼容写法，并明确说明该 SVG 是纯矢量还是位图容器。
- 压缩或改写论文中含数学公式的段落时，严禁用 Eq. 编号或占位符替代公式本体；必须明确保留原公式，不删除、不重写、不转写。
- 论文正文中的图表必须先在正文中明确引用，再放置对应图表；不得出现缺少正文引用的图表或表格。
- 今后处理用户原始文件时，不直接覆盖原文件；必须先新建副本并在副本上修改，除非用户明确要求覆盖原文件。任务完成后必须清理本次任务产生的临时文件；若临时文件包含用户原始资料或删除风险不明确，则先列出清单并征得确认。
- 生成论文图表时，如果用户说明已有预测结果，不得重新推理；必须基于已有 CSV/JSON、原图、GT 和预测图重绘。所有论文图片需同时输出 PNG、SVG、PDF，并保存本次生成脚本、配置和 manifest。
- 用户将 GroupA 作为毕业设计项目改名为 SegSAM 时，不需要剥离或重构联邦学习源码；只做项目命名、提交叙事、图表复现与必要提交材料整理。
- 向用户说明技术方案时必须简洁、直接，并严格保持用户原始目标，不得擅自扩展或改写研究目的。
- 判断本机是否具备 PyTorch、CUDA 或其他运行环境时，必须检查全部相关 Conda/虚拟环境和硬件状态，不得仅依据默认 Python 环境下结论。
- 未经用户明确要求，不得在本地启动训练；用户要求项目审查时仅执行只读检查，并将实际训练与 smoke run 留在目标服务器。
